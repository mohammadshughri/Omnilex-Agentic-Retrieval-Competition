Architecture V2

The 4-Layer Pipeline

Layer 1 Regex Extraction Pull Art. citations directly from query text using improved patterns + FR→DE mapping In progress

Layer 3 LLM Reasoning ★ Given query + candidates → reason like a lawyer → output final citation list Needs API docs

Each layer is verified on val.csv before the next is built.  No guessing, no wasted work.

Layer 2 · Already built

Dense Retrieval — one change

BGE-M3 ✓

What we keep

  • • BGE-M3 multilingual embeddings
  • • FAISS index on laws + courts corpora
  • • DenseRetriever + DenseIndex pipeline

The one change: k=10 → k=25

Oracle F1 at k=100.644
Oracle F1 at k=250.779
Gain+13.5% oracle ceiling

Why k=25?  Val queries have mean 25.1 gold citations (range 10–47). Submitting only k=10 means we miss 60% of citations on average before even scoring.

68% of val gold BGE citations already exist in the court corpus → retrieval CAN find most of them.
